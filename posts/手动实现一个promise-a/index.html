手动实现一个promise/A+

December 3, 2018 7 minutes read

开始

用了这么久Promise，今天突然想研究下Promise是怎么实现的，并且自己实现一个。

准备工作

既然要手动实现一个Promise,那么先看看我们经常使用的Promise是什么样子的；

使用样例

let promise = new Promise(function(resolve,reject){
    resolve("success"); 
})
promise.then(function(value){

}, function(reason){

})

从样例上看，Promise是一个构造函数，并且有一个参数excutor，这个参数是一个函数，也有两个函数作为参数，resolve和reject，resolve和reject是Promise内部的两个函数 then方法是原型链上的方法，接收两个入参（函数），这两个函数分别对应Promise的变更为resolve状态和reject状态后要执行的操作；

除了样例，我们还应该了解下Promise/A+规范。

Promise/A+规范

全文翻译：Promise/A+规范

Promise状态

一个promise必须处于一下三种状态：pending、fulfilled或者rejected。

  • pending状态：promise可以转成fulfilled和rejected状态
  • fulfilled状态：promise的状态不可变更，并且必须有一个值，且这个值不能被改变；
  • rejected状态： promise的状态不可变更，并且必须有一个原因，且这个原因不可改变；

then方法

promise必须包含一个then方法来访问它当前或者最终的值或者原因。 Promise的then方法接收两个参数：promise.then(onFulfilled, onRejected)

  • onFulfilled和onRejected必须是一个函数；
  • onFulfilled在promise到fulfilled状态后触发，promise的值是它的第一个参数。（onFulfilled只能触发一次，且在promise状态变成fulfilled之前不能触发）
  • onRejected在promise到rejected状态后触发，promise的原因是它的第一个参数。（onRejected只能触发一次，且在promise状态变成rejected之前不能触发）
  • then方法可能在相同的promise中被调用多次。onFulfilled和onRejected必须按照原有顺序进行调用执行；

这里先列举基础的部分；

promise实现—part 1

从上面的信息我们可以模拟出Promise的基本结构

const PENDING = "pending";
const RESOLVED = "fulfilled";
const REJECTED = "rejected";
function Promise(excutor){
    const that = this;
    that.status = PENDING;    // 初始状态为pending
    that.value = undefined;   // 成功后的返回值
    that.reason = undefined;  // 失败的原因
    that.onFullfilledList = [];  // 存放onFullfilled函数的列表，当promise处于pending状态时，多次调用then，会有多个onFullfilled函数待执行
    that.onRejectedList = [];   // 存放onRejected函数的列表，当promise处于pending状态时，多次调用then，会有多个onFullfilled函数待执行
    function resolve(value){
        if(that.status === PENDING) {
            that.status = RESOLVED;
            that.value = value;
            that.onFullfilledList.forEach(function(fullfilled){
                fullfilled(value);
            })
        }
    }
    function reject(reason){
        if (this.state === PENDING) {
            that.status = REJECTED;
            that.reason = reason;
            that.onRejectedList.forEach(function(rejected){
                rejected(reason);
            })
        }
    }
    try{
        excutor(resolve, reject)  //立刻执行传入的函数
    }catch(err){
        reject(err)  // 执行过程抛出异常，则以异常原因拒绝promise
    }
}
Promise.prototype.then = function(onFullfilled, onRejected) {  // then方法在原型链上
    if(this.status === RESOLVED && typeof onFullfilled === "function"){
        onFullfilled(this.value);  // fulfilled状态则立即执行onFullfilled函数
    }
    if(this.status === REJECTED && typeof onRejected === "function"){
        onRejected(this.reason);  // rejected状态则立即执行onRejected函数
    }
    if(this.status === PENDING){
        if(typeof onFullfilled === "function"){
            this.onFullfilledList.push(onFullfilled);  // 注册onFullfilled
        }
        if(typeof onRejected === "function"){
            this.onRejectedList.push(onRejected);     // 注册onRejected
        }
    }
}

这样基本满足了上面所列的规范中的基础点，但规范中的要求还有很多；

promise实现—part 2

规范中还对then方法有约束：

  • then方法必须返回一个promise： promise2 = promise1.then(onFulfilled, onRejected); *promise可以链式调用的关键
  • 如果onFulfilled或者onRejected方法返回一个值x，那么执行promise的解析过程 解析函数规范-2.3章节
  • 如果onFulfilled或者onRejected方法抛出一个异常e，promise2必须使用e作为原因拒绝掉（rejected）。
  • 如果onFulfilled不是一个函数并且promise1到了fulfilled状态，那么promise2必须在与promise1的值相同的情况下转换到fulfilled状态。
  • 如果onRejected不是一个函数并且promise1到了rejected状态，那么promise2必须在与promise1的原因相同的情况下转换到rejected状态。

先忽略解析过程，实现其他规范要求：

Promise.prototype.then = function(onFullfilled, onRejected) {
    let that = this;
    let promise = new Promise(function(resolve, reject){
        // 把onFullfilled和onRejected不是函数的情况单独处理，减少if..else
        onFullfilled = typeof onFullfilled === 'function' ? onFullfilled : function(value){resolve(value)}
        onRejected = typeof onRejected === 'function' ? onRejected : function(reason){reject(reason)}
        try{
            if(that.status === RESOLVED){
                let x = onFullfilled(that.value);  // fulfilled状态则立即执行onFullfilled函数
                // TODO 解析函数
                analysisPromise(promise, x, resolve, reject) //这里直接传入promise会抛异常，因为promise并没有定义
            }
            if(that.status === REJECTED){
                let x = onRejected(that.reason); 
                // TODO 解析函数
                analysisPromise(promise, x, resolve, reject)
            }
            if(that.status === PENDING){
                that.onFullfilledList.push(function(){// 这里需要在onFullfilled函数之外包一层
                    let x = onFullfilled(that.value);
                    // TODO 解析函数
                    analysisPromise(promise, x, resolve, reject)

                that.onRejectedList.push(function(){
                    let x = onRejected(that.reason);  
                    // TODO 解析函数

                });
            }

            reject(err)
        }
    })
    return promise
}

promise实现—part 3

在Promise/A+规范中，解析函数是这样规定的：解析函数规范-2.3章节

内容有点多，我试着按自己的理解总结一下：
解析函数需要两个入参，一个then方法即将作为返回的promise，一个是onFullfilled或onRejected返回的对象x;
然后，对x进行处理：

  • 情况一： 当x和promise指向同一对象，则抛出异常；因为这样会造成循环引用；例如：
let p = new Promise(resolve => {
  resolve(0);
});
var p2 = p.then(data => {
  // 循环引用
  return p2;
})

  • 情况二： 当x不是一个对象或函数时，那么就用x完成promise；

  • 情况三， 如果x是一个promise，就判断它的状态
    这里有个问题，怎么判断x是一个promise呢？这里实际上js中是没有办法准确判断x是否是promise对象，只能假定x具有和promise类似的行为，也就是规范中所说的thenable对象，尝试将promise转换为x的状态；如果失败就reject
    这就是下面的情况四；

  • 情况四， 如果x是一个对象或者函数，规范中明确规定要*then = x.then

    • then不是函数，就直接用x完成promise；*x不是一个thenable对象
    • 检测then的过程抛出异常，那就把这个异常当做原因拒绝promise； *then函数不符合promise规范
    • 如果then是一个函数，那么用x作为this来调用它，第一个参数是resolvePromise，第二个参数是rejectPromise：*用call执行then，指定this为x
    • 如果resolvePromise被值y调用，那么运行[Resolve]。*如果x被解决（resolve），并且值为y，也就是执行了resolvePromise，那么就对y继续进行解析；重复上面流程；
    • 如果rejectPromise被原因r触发，那么用r来拒绝promise。*如果x被拒绝（reject），并且原因是r，也就是执行了rejectPromise，那么就用r把promise拒绝；
    • 如果resolvePromise和rejectPromise都被调用，或者使用相同的参数多次调用，那么第一次调用生效，其他之后的任何调用都忽略掉。*resolvePromise和rejectPromise只能被调用一次
    • 如果在调用then方法时抛出了一个异常e,如果resolvePromise和rejectPromise已经被调用了那么就忽略掉它。否则，使用e作为原因拒绝promise。*catch的异常要根据resolvePromise和rejectPromise的调用情况处理

从上面的规范可以看出来，解析函数的重点是为了解决onFullfilled或onRejected中return 另一个promise对象的情况； 例如：

let p = new Promise((resolve, reject) => {
  reject(0);
});
var p2 = p.then(data => {
    let p3 = new Promise((resolve, reject) => {
        reject(99);
    })
    return p3;
}, err => {
    let p4 = new Promise((resolve, reject) => {
        resolve(88);
    })
    return p4
})
console.log(p2)//  fulfilled  88

因为then方法最终要返回一个新的promise
如果onFulfilled, onRejected回调函数中返回值，或者不是thenable对象，那么就以x为值变更为fufilled状态；
如果onFulfilled, onRejected回调函数中返回promise2（或thenable函数），那么调用promise2的then方法，并根据将promise转换成promise2的状态；
如果promise2的回调函数中仍然返回新的promise对象，则递归执行上一步，直到then方法的回调函数返回的不在是promise；

实现如下：

function analysisPromise(promise, x, resolve, reject){
    if(promise === x) {
        return reject(new TypeError("循环引用"));
    }
    if(x === null){
        return resolve(x)
    }
    if(typeof x !== "object" && typeof x !== 'function'){
        return resolve(x);
    }
    let called;
    let then = x.then
    if(typeof then !== "function"){
        return resolve(x);
    }
    try{
        then.call(x, function(y){
            if(called)return;
            called = true;   
            analysisPromise(promise, y, resolve, reject);
        }, function(r){
            if(called)return;
            called = true;
            reject(r)
        })
    }catch(e){
        if(called)return;
        called = true;
        reject(e);
    }
}

class Promise {
  constructor(executor) {
    this.status = 'pending'; // 初始状态为pending
    this.value = undefined;
    this.reason = undefined;
    this.onFullfilledList = []; // 存放onFullfilled函数的列表，多次调用then，会有多个onFullfilled函数
    this.onRejectedList = []; // 作用同上
    const resolve = value => {
      if (this.status === 'pending') {
        this.status = 'fulfilled';
        this.value = value;
        for (let fullfilled of this.onFullfilledList){
          fullfilled();
        }
      }
    }
    const reject = reason => {
      if (this.status === 'pending') {
        this.status = 'rejected';
        this.reason = reason;
        for (let rejected of this.onRejectedList){
          rejected();
        }
      }
    }
    try {
      executor(resolve, reject)
    } catch (err) {
      reject(err) 
    }
  }

              reject(e);
            }

              // TODO 解析函数
