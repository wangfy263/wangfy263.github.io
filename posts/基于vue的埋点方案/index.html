基于vue的埋点方案

May 7, 2018 One minute read

buried-point基于vue的埋点方案

安装Install

npm install buried-point

配置configuration

main.js中进行如下配置： 


  import BuriedPoint from ‘buried-point’



let option = {
    router: router,  //vue路由对象
    domain: ‘127.0.0.1:18080’,  //埋点服务器地址
    path: ‘/nresource/dcsImg’   //埋点请求路径  /nresource/dcsImg/dcs.log
  }
  Vue.use(BuriedPoint, option)

开始使用start

1.v-ready-stat指令，用于采集页面加载完成时数据 在需要采集数据的vue组件根节点上声明埋点指令v-ready-stat


  <template>
  <div class="button-content" v-ready-stat>
  </template>

如果需要在页面加载时传递数据，v-ready-stat指令可以接收json格式的参数{'SI.si_n':'WEB', 'SI.si_x':'002'}


  <div class="icon-content" v-ready-stat="{'SI.si_n':'xxx','SI.si_x':'009'}">

2.v-click-stat指令，用于采集用户点击事件的数据 在被点击的dom节点上声明埋点指令v-click-stat,数据格式{'SI.si_n':'WEB', 'SI.si_x':'002'}


    <el-button type="text" v-data-stat="{'SI.si_n':'WEB', 'SI.si_x':'002'}">{{message}}</el-button>

3.命令式采集业务码 针对需要在业务逻辑中进行埋点的特殊场景，提供命令式埋点方案； 在vue组件中需要import埋点插件import bp from 'buried-Point',在需要埋点的位置调用埋点方法bp.dcsMultiTrack('SI.si_n', 'WEB_Q', 'SI.si_x', '005')


import bp from 'buried-Point'
export default {
  //在vue组件的生命周期钩子mounted阶段，进行埋点
  mounted() {
    bp.dcsMultiTrack('SI.si_n', 'WEB_Q', 'SI.si_x', '005')
  }
}
